# Mosaic v0.3 — Interactive Operator Demo Handoff

This is the builder handoff for the interactive, operator-driven demo increment.
It implements [`RFC-0005`](../rfcs/RFC-0005-interactive-operator-simulation.md)
and the recorded [`demo-script.md`](../demo-script.md). The completed v0.2 board
remains in [`HANDOFF.md`](../../HANDOFF.md).

## Increment goal

Turn the startup-only fixture demo into an interactive session a real operator
drives: **Start simulation** creates a synthetic session, events arrive in
configured timing, the operator triggers Analyze/briefing and records handoffs,
and a provenance/actions tab shows the full trail. Models may run live against
OpenAI, opt-in and budgeted, with the deterministic fixture path as the default
and fallback. Recurrence awareness surfaces prior area handoffs and prepares
reviewable notes — never autonomous contact, never LLM self-healing.

## Branch and worktree model

Integration branch: `main`. Every parcel uses an isolated branch and worktree
from the latest integrated `main` SHA recorded by the coordinator:

```text
main
├─ parcel/P35-interactive-contracts
├─ parcel/P36-live-model-contracts
├─ parcel/P37-operator-session
└─ … one isolated worktree per claimed parcel
```

Do not reuse another parcel's worktree. Claim a row only when all its
prerequisites are `✅ Integrated`. Builders do not integrate their own work.

## Non-negotiable invariants (every parcel)

- Synthetic data only; no real feeds, PII, secrets, or model binaries in Git.
- The **default and fallback path is the deterministic fixture**; the demo must
  pass with no live secret present.
- Live model output **informs**; it never mutates the deterministic projection
  or issues an operational action. Invalid/refused/unavailable output is a
  recorded ModelRun that changes no state.
- Every operator action is an immutable audit record with `executed: false`.
- The OpenAI key is a **server-only** runtime secret; never in the browser, Git,
  image, or committed compose file.
- Reuse the RFC-0004 profile seam. Core packages stay free of domain identifiers.

## Per-parcel briefs

Each parcel is atomic: exclusive path ownership and an independent acceptance
check. Keep changes additive; do not alter frozen datasets, ontology schemas,
generated ontology code, migrations, or module dependencies unless the parcel
explicitly owns them.

### P35 — interactive contracts
Add additive interfaces/values in `internal/contracts` for: an operator session
actor (identity, role, labels); an interactive simulation session (session id,
status `pending|running|ended`, ordered beats with a configured relative
schedule); and a session-scoped stream event. No behavior, no HTTP, no storage.
Regenerate the checked-in GoMock. **Accept:** contract + mock tests compile and
prove the surface; `go vet` + `go test ./internal/contracts/...` + `go run ./cmd/mosaic quality` pass.

### P36 — live-model contracts
Add additive contracts for per-agent provider selection (`fixture` vs `live`) and
budget/usage records (input/output tokens, estimated cost, per-session counts).
No provider implementation. Regenerate GoMock. **Accept:** as P35.

### P37 — operator session
Implement a server-side operator login/session store and an `api.ActorResolver`
adapter that resolves the logged-in operator into the audit actor. Preserve the
public read policy defaults; no password/PII/external auth. **Accept:** focused
tests prove login→session→actor resolution and that unauthenticated reads keep
the public default; `go test ./internal/operator/...` + quality pass.

### P38 — interactive simulation controller
Core, domain-agnostic lifecycle: start (new session, clear visible workspace
signal), replay beats in configured timing on a session-scoped stream, explicit
end, and reset-to-new-session that never rewrites immutable history. Timing is
injected configuration. Consumes the profile beat schedule via an interface, not
a domain import. **Accept:** tests prove start/beat-order/end/reset, that a new
session appends no duplicate immutable records, and deterministic timing with an
injected clock; `go test ./internal/simsession/...` + quality pass.

### P39 — domain beat schedule
The domesticdisturbance profile exposes its configured-timing interactive beat
sequence derived from the frozen scenario (no new dataset). **Accept:** tests
prove the beat order/timing matches the scenario and revisions 7/9 remain the
advisory pivots; package tests + quality pass.

### P40 — OpenAI transport
Implement Luna/Terra/Sol structured clients over the OpenAI Responses API behind
their existing seams. Opt-in and per-agent; server-only `OPENAI_API_KEY` read at
runtime (no downloader, no key in code/config); deterministic fixture client used
when live is not selected or the secret is absent. Refusals/invalid/unavailable
responses map to recorded ModelRuns with no state mutation. **Accept:** tests use
a stubbed HTTP transport (no network); prove request shaping, refusal/timeout
handling, and fixture fallback; no secret in the repo; quality passes.

### P41 — budget governor
Enforce an application budget cap, per-response output-token caps, and per-session
model-call limits; write usage/audit records. Exceeding a limit fails closed
(records the attempt, returns no advice) rather than silently overspending.
**Accept:** tests prove cap/limit enforcement and usage recording; quality passes.

### P42 — simulation-session API
Add start/reset/status routes and a session-scoped SSE stream over the P38
controller. Preserve existing routes, bounded serialization, and raw-payload
exclusion. **Accept:** API tests prove session start/stream/end and no-op reset
safety; `go test ./internal/api/...` + quality pass.

### P43 — operator action API
Add Analyze(Terra)/Brief(Sol)/Interpret(Luna) and approve/annotate/prepare-handoff
endpoints. Each write is an immutable `executed:false` audit record carrying the
operator actor. Model calls route to fixture or live per config and pass through
the budget governor. **Accept:** tests prove executed:false writes, operator
actor propagation, fixture-and-live routing (live via stub transport), and
budget denial; quality passes.

### P44 — recurrence awareness
Deterministically detect prior area-specific recorded handoffs and, after a
configurable window, prepare a pending reviewable note linked to the prior
immutable records. No autonomous contact; the note is `executed:false`.
**Accept:** tests prove area matching, the configurable window, prepared-note
linkage, and that nothing is "sent"; `go test ./internal/recurrence/...` + quality.

### P45–P48 — UI (serial on `ui/**`)
Build the incident-first workspace with **Start simulation** and **Analyze**
(P45); recipient action cards + annotation/decision controls (P46); the
provenance/actions tab (P47); and the recurrence surface + live/fixture indicator
(P48). No raw-payload or model-response leakage; explicit recorded-vs-delivered
language; keep developer diagnostics in a compact drawer. **Accept per parcel:**
`npm run check`, `npm run build`, and `go run ./cmd/mosaic quality` pass; Svelte
checks cover the parcel's states.

### P49 — executable composition
Wire operator session, sim controller, live/fixture model selection, budget, and
recurrence into `cmd/mosaicdemo` startup. Server-only key handling; fixture path
when no secret. Do not change UI/API/simulator/contracts source. **Accept:**
package tests prove fixture startup unchanged and live selection is chosen only
when explicitly configured with a secret; quality passes.

### P50 — acceptance
Interactive E2E (start→beats→Analyze→handoff→provenance), a Docker runbook
update, and `docs/live-models.md` documenting opt-in, server-only key, budget,
and fixture fallback. Exercise the image/compose without modifying them.
**Accept:** E2E + Svelte checks + full quality + a fresh isolated Docker
build/start/smoke, each reported verbatim.
