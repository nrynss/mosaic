# Mosaic synthetic dataset generator

Produce data for a fictional demonstration only. Every person, address, phone
number, vehicle, agency, incident, unit, road, and weather observation must be
invented. Do not reproduce, infer, or request real-world operational data or
personally identifiable information.

Return exactly one JSON object and nothing else: no Markdown fence, commentary,
analysis, explanation, or operational advice. The object must have this exact
shape:

```json
{
  "bundle_version": "1.0.0",
  "scenario_id": "<the supplied scenario_id>",
  "artifacts": {
    "manifest.json": {"...": "DatasetManifest-conforming object"},
    "scenario.json": {"...": "Scenario-conforming object"},
    "raw-events.json": {"...": "raw event document"},
    "expected-outcomes.json": {"...": "expected outcomes document"}
  }
}
```

The four artifact values must be JSON objects, not quoted JSON strings. Conform
exactly to every read-only Mosaic schema supplied after this instruction. Do not
add fields, invent schemas, revise schema versions, or explain schema design.
Use all required schema versions in the manifest and make every reference,
checksum, evidence citation, ID map, and correction relationship internally
consistent.

Build a bounded domestic-disturbance-style synthetic scenario with reported
facts only in source events, derived assessments only in insights, and neutral
human-review-only recommendations. Include evidence and provenance required by
the schemas. Never recommend dispatching, contacting, detaining, surveilling,
or otherwise acting on a person. The generated result is a candidate for human
review and schema validation; it is not an operational instruction.
