# Mosaic synthetic dataset generator — complete-schema revision

Produce data for a fictional demonstration only. Every person, address, phone
number, vehicle, agency, incident, unit, road, and weather observation must be
invented. Do not reproduce, infer, or request real-world operational data or
personally identifiable information.

Return exactly one JSON object and nothing else: no Markdown fence, commentary,
analysis, explanation, or operational advice. The object must have this exact
shape:

```json
{
  "bundle_version": "1.0.0",
  "scenario_id": "<the supplied scenario_id>",
  "artifacts": {
    "manifest.json": {"...": "DatasetManifest-conforming object"},
    "scenario.json": {"...": "Scenario-conforming object"},
    "raw-events.json": {"...": "raw event document"},
    "expected-outcomes.json": {"...": "expected outcomes document"}
  }
}
```

The four artifact values must be JSON objects, not quoted JSON strings. Conform
exactly to every read-only Mosaic schema supplied after this instruction. Do not
add fields, invent schemas, revise schema versions, or explain schema design.

The manifest `prompt_version` must be `2.0.0`. Its `id_map` must cover every
identifier referenced by all four artifacts. Every evidence citation,
correction/supersession link, source-record reference, checksum, and timestamp
must be internally consistent with the other artifacts.

## Synthetic scenario requirements

Build a bounded domestic-disturbance-style synthetic scenario with several
time-ordered source events. Include a fictional initial report, a welfare or
status update, a neutral resource or road update, and at least one explicit
correction that supersedes an earlier incomplete or mistaken record. Facts may
appear only in source/canonical records; derived assessments only in insights;
recommendations must be neutral and human-review-only. Include the evidence,
provenance, and audit history required by the schemas.

Never recommend dispatching, contacting, detaining, surveilling, or otherwise
acting on a person. The generated result is a candidate for human review and
schema validation; it is not an operational instruction.
